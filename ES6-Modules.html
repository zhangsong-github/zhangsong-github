何为模块

模块是采用不同方式加载的 JS 文件（与原先的脚本加载方式相对）。这种种不同模式很有必要，因为它与脚本有大大不同的语义。

  1. 无法退出的严格模式。
  2. 在模块的顶级作用域变量，不会自动添加到共享的全局作用域，它们只会在模块的顶级作用域存在。
  3. 模块顶级作用域的 this 值为 undefined。
  4. 模块不允许在代码中使用 HTML 风格的注释。
  5. 对于需要让模块外部代码访问的内容，模块必须导出它们。
  6. 允许模块从其他模块导入绑定。

基本的导出

你可以使用 export 关键字将已发布代码部分公开给其他模块。最简单的方法就是将 export 关键字放在被公开的变量声明前面。

基本的导入

import 语句有两个部分，一是需要导入的标识符，而是需要导入的标识符的来源模块。

当从模块导入了一个绑定时，该绑定表现得就像是使用了 const 定义。你不能在定义一个同名变量。也不能在 import 语句之前使用此标识符。
